AAA Testing

AAA is an acronym that stands for Arrange / Act / Assert. It is a pattern for arranging and formatting code in Test methods. The idea is to develop a unit test by following these 3 simple steps:

  • Arrange – setup the testing objects and prepare the prerequisites for the test.
  • Act – perform the actual work of the test.
  • Assert – verify the result.

For the Calculator Program , AAA testing proves to be very useful as testing provides a feedback loop for development. That’s how testing keeps us safe. With tests, we know when things break.

"Pytest" if the test framework used in python to to automate tests, execute suites, and report results

I have demonstrated 2 types of testing done on the calculator program below.

Unit Testing
The code below shows a code snippet of AAA testing done on the add method of calculator class.

Example 1:
        def num_list():
             """Arranging for AAA Testing"""
                return 1.0, 2


        def testing_add_method():
            """Act for AAA testing"""

            result = CalculatorMain.add(num_list())

            """Assert for AAA testing"""

The comments show each phase of testing.

  • The Arrange step returns a list named "num_list" with values 1.0 and 2 for testing.
  • The Act step calls the “add” function using the “num_list” list and stores the returned value in a variable named “result”
  • The Assert step verifies that “result” is equal to 3 (1.0+2) value.

Feature Testing
Here i am explaining a test case from the simple_pages_test. The code below shows a code snippet of AAA testing done to check the git page of my website using client.get

Example 3:
                def test_request_git_page(client):
                    """This tests the git page"""
                    """ The Arrange phase"""
                    url_to_request = "/git"
                    """Act Phase"""
                    response = client.get(url_to_request)
                    """ The Assert Phase"""
                    assert response.status_code == 200
                    assert b"GIT" in response.data

  • The Arrange step passes the url ending with “git” from the website.
  • The Act steps makes a GET request to the /git endpoint which returns the content of the git webpage.
  • The Assert steps then verifies that the HTTP status code was 200, meaning “OK” or “success,” and that the word “GIT” appears somewhere in the response’s abstract text.

AAA testing forces tests to focus on independent, individual behaviors. It separates setup actions from the main actions and subsequent actions and assertions belong in separate tests
