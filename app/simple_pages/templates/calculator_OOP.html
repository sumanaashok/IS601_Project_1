Demonstrating OOP Principles Using Calculator Class

In this article i have explained the 4 principles of object oriented programming : Inheritance , Polymorphism , Abstraction and Encapsulation using examples from the calculator program.

Inheritance
Consider the example below 

                class Summation(Calculator):
                    """addition class"""

                    def calculate_res(self):
                        """returns addition results"""
                        sum_values = 0.0
                        for num in self.values_list:
                            sum_values = Add.add(num, sum_values)
                        return sum_values

The Summation class is extending the abstract Calculator class. The summation class inherits the methods "create" and "float_conversion" and also the list "num_list" from calculator This way , these methods need not be written in summation class again and hence provides re-usability of code

Polymorphism
The below example illustrates polymorphism 

                        @staticmethod
                        def add(num_list):
                            """ This is the add method"""
                            calculation = Summation.create(num_list)
                            return calculation.calculate_res()

In this example , the "calculate_res" method can be used by any object of the "calculator" type , and we can call it on any class on the "Calculator" , that is "Summation" , "Difference" , "Product" or "Divide" and we will get the appropriate result back.

Encapsulation
Consider the below example 

                 @staticmethod
                def multiply(num_list):
                    """ This is the multiplication method"""
                    calculation = Product.create(num_list)
                    return calculation.calculate_res()

Here , "calculation" variable holds/stores an instance of the class "Product". it encapsulates the data that is the list "num_list" and the functionality to work on that data that is the method used to multiply the numbers in the list "calculate_res" to get the result.

Abstraction
Consider the below example 

                class CalculatorMain:
                    @staticmethod
                    def subtract(num_list):
                        """ This is the subtract method"""
                        calculation = Difference.create(num_list)
                        return calculation.calculate_res()

                 class Difference(Calculator):
                 """subtraction class"""

                         def calculate_res(self):
                            """returns the subtraction results"""
                            diff_values = 0.0
                            for value in self.values_list:
                                diff_values = Sub.sub(diff_values, value)
                            return diff_values

                    class Sub:
                    """ This class is used for subtraction of 2 numbers"""

                            # pylint: disable=redefined-outer-name
                            @staticmethod
                            def sub(num1, num2):
                                """This method does the subtraction operation"""
                                sub_res = num1 - num2
                                return sub_res

These 3 methods demonstrate the principle of "Abstraction". first the method "subtract" of the "CalculatorMain" class is calling the method "calculate_res" of the "Difference" calculator class next , the "calculate_res" method is calling the "sub" method of the "Sub" class inside the loop. finally the "sub" method is doing the subtraction calculation. All this underlying functionality is hidden when the "calculate_res" method is called, showing how abstraction reduces the complexity of the program.
