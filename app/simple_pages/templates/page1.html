GIT

Branches
  • Branches in Git allows us to develop new features or add new features to the existing code , correct any errors and to safely carry out experimental work on our repository.
  • Helps us separate the new development work without affecting other branches.
  • Branches are used to keep the master/base branch working and clean of errors.
  • “git checkout -b” command is used to create new branches
  • Branches can be created from existing branches
  • Once the changes in a branch are tested and ready to be deployed , then a pull request is opened to merge changes into the base/master branch.
  • After a pull request has been merged, or closed, you can delete the head branch as this is no longer needed.
  • When there is an open pull request then that branch cannot be deleted.
  • We can have branches that are protected. Delete or force push to the branch on such branches cannot be done if you are not the repository administrator.
  • Repository administrator can also be added to follow the rules on protected branches.

Merge
  • Git merge combines sequences of commits into one single history of commits.
  • In most cases, git merge is used to combine two branches
  • We can use git checkout to switch to the branch we want to merge into , when we need to combine branches locally. This branch is typically the master/main branch.
  • We can use git merge and specify the name of the other branch to bring into this branch.
  • Merge Incorporates changes from the named commits (since the time their histories diverged from the current branch) into the current branch.

Commit
  • Used to record the changes made to a repository.
  • Can be viewed as snapshot of the project's currently staged changes. Committed snapshots can be thought of as safe versions of a project.
  • By using git add command we can specify the content to be committed
  • The -m option is used to specify a message for the commit.
  • Once the changes are committed , they can be pushed to the remote repository from local repository.

Tags
  • Tags are used to mark a particular in a repository’s history as important.
  • They are be used to mark different versions of the same product.
  • git tag lists all the tags of the repository.
  • There are 2 types of tags light weight and annotated.
  • Annotated tags store complete information like name, email of the tagger , date , checksum etc.
  • Lightweight tags only store the checksum and are usually used as temporary tag.

Repository
  • A Git repository tracks and saves the history of all changes made to the files in a Git project. It saves this data in a directory called . git , also known as the repository folder. Git uses a version control system to track all changes made to the project and save them in the repository.
  • Git repository can be created by either cloning an existing git repo from elsewhere or by taking a local directory that is currently not under version control, and turn it into a Git repository.

What is GIT?

Git is a DevOps tool used for source code management. It is a free and open-source version control system used to handle small to very large projects efficiently. Git is used to tracking changes in the source code, enabling multiple developers to work together on non-linear development.
